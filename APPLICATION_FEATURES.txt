================================================================================
  RAILWAYPREP — COMPLETE APPLICATION FEATURES
================================================================================

TARGET AUDIENCE : Railway exam aspirants (RRB NTPC, Group D, JE, ALP, Technician)
TECH STACK      : React 19 + TanStack Router + TanStack React Query + Tailwind CSS
API BASE URL    : https://railwayprep-repo.onrender.com/api/v1
AUTH METHOD     : Bearer token stored in cookie (rrb_token)

================================================================================
1. AUTHENTICATION
================================================================================

LOGIN PAGE (/login)
  - Email + password fields with password visibility toggle
  - Form validation with inline error messages
  - Loading state on submit
  - Redirect to dashboard on success
  - Link to registration page

REGISTRATION PAGE (/register)
  - Fields: Full Name, Email, Password (min 6 chars), Category, Target Exam
  - Category options: UR, SC, ST, OBC, EWS
  - Category benefit banner explaining age relaxation, fee waiver, lower cutoffs
  - Target exam dropdown: RRB NTPC, RRB Group D, RRB JE, RRB ALP, Technician
  - Error handling and loading state
  - Stores access token in cookie on success

LOGOUT
  - Available from sidebar bottom (with confirmation)
  - Clears rrb_token cookie and redirects to login

================================================================================
2. NAVIGATION & LAYOUT
================================================================================

SIDEBAR (Desktop)
  - Collapsible: 240px expanded / 64px collapsed (icon-only)
  - Logo and "RailwayPrep" branding
  - Displays logged-in user name and target exam (hidden when collapsed)
  - Blue gradient background with active-route highlight
  - Section — Preparation:
      Dashboard, Mock Tests, PYQ Bank, Notifications,
      Study Planner, SC/ST/OBC Guide, Analytics, Leaderboard, Subscription
  - Section — Management:
      Admin Panel (visible only to admin users)
  - Logout button at bottom

MOBILE NAVIGATION
  - Top bar with hamburger menu icon and RailwayPrep logo
  - Slide-out overlay sidebar with same navigation items

================================================================================
3. DASHBOARD (/dashboard)
================================================================================

  - Welcome banner with user's name, average score, and current rank
  - Upgrade prompt banner for free-plan users (₹199/mo premium offer)
  - Quick Stats cards:
      Tests Taken | Avg Score | Rank | Target Exam
  - Weekly Activity line chart (questions solved per day)
  - Quick Access buttons linking to: Tests, PYQ Bank, Analytics, Planner, Leaderboard
  - Recent Tests section:
      List of last attempted tests with best score badges and "Retake" button
  - Latest Notification banner at bottom

================================================================================
4. MOCK TESTS (/mock-tests)
================================================================================

TEST LISTING PAGE
  - Responsive grid (1 / 2 / 3 columns)
  - Exam filter tabs: All | RRB NTPC | RRB Group D | RRB JE | RRB ALP
  - Difficulty filter tabs: All | Easy | Medium | Hard
  - Completion counters: X Completed, Y Pending
  - Loading and error states
  - Empty state when filters match nothing

TEST CARD (per test)
  - Title with completion checkmark if already attempted
  - Badges: Exam type, Difficulty, Negative Marking indicator
  - Stats: Question count, Duration (minutes), Total Attempts
  - Best score badge (shown only if previously attempted)
  - "Start Test" or "Retake Test" action button

TEST-TAKING INTERFACE (/take-test/:testId)
  - Full-screen layout with top bar showing:
      Test title, exam type, answered/skipped count, countdown timer
  - Timer turns red when less than 5 minutes remain
  - Auto-submit when timer reaches 0:00
  - Flag button to mark questions for review
  - Question display:
      Subject and topic label, question number, question text
  - Multiple-choice options (A/B/C/D) with visual selection state
  - Navigation: Previous | Skip (jumps to next unanswered) | Next
  - Question Navigator sidebar (right):
      Grid of question buttons color-coded by status —
        Blue = Current, Green = Answered, Orange = Flagged, Gray = Not Visited
      Summary: answered / unanswered / flagged counts
  - Submit button opens confirmation modal with attempt summary

RESULT VIEW (after submission)
  - Percentage score with grade label:
      Excellent! (≥80%) | Good Job! (≥60%) | Keep Going! (≥40%) | Needs Improvement (<40%)
  - Stats grid: Score, Correct, Wrong, Skipped
  - Time taken and accuracy percentage
  - Full answer review for every question:
      Correct answer highlighted green, user's wrong answer shown in red, skipped in gray
      Explanation shown for each question
  - Action buttons: All Tests | View Analytics

12. ADMIN PANEL (/admin)  — admin users only

4-TAB INTERFACE:

TAB 1 — OVERVIEW
  - Stats cards: Total Users | Mock Tests | Questions in Bank | Active Subscriptions
  - Recent Activity timeline: action type, detail, timestamp

  - Searchable user table (filter by name or email)

13. API SURFACE SUMMARY
================================================================================

AUTH          POST /auth/login, /auth/register
              GET  /auth/me
              PATCH /auth/me (profile update)
              POST /auth/logout

TESTS         GET  /tests?exam=&difficulty=&limit=
              GET  /tests/:id
              GET  /tests/:id/review

QUESTIONS     GET  /questions?subject=&difficulty=&isPYQ=&page=&limit=
              GET  /questions/pyq-topics

ATTEMPTS      POST /attempts/:testId/submit
              GET  /attempts
              GET  /attempts/:id

ANALYTICS     GET  /analytics

LEADERBOARD   GET  /leaderboard

NOTIFICATIONS GET  /notifications
              GET  /notifications/:id

SUBSCRIPTIONS GET  /subscriptions/plans
              GET  /subscriptions/my
              POST /subscriptions/order
              POST /subscriptions/verify

PLANNER       GET  /planner
              GET  /planner/stats
              POST /planner
              PATCH /planner/:id
              DELETE /planner/:id

ADMIN         GET  /admin/dashboard
              GET  /admin/stats
              GET  /admin/users
              PATCH /admin/users/:id/deactivate
